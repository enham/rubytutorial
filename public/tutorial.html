Tutorial

Code for this demo can be found at https://github.com/enham/rubydemo. Make sure to bundle install to resolve any platform dependencies before pushing to OpenShift.


Pre-requisites: The following software should be installed and available in your local environment:

  • Ruby 1.9.3
  • Rails 3.2.x (if rvm is available, use rvm use ruby-1.9.3 command to manage versions and dependencies)
  • rhc (gem install rhc)

Also, please note that many aspects of Ruby are platform dependent. If you are using Windows, it may implicitly declare gems in your Gemfile.lock that are Windows-specific and some gems may not work altogether. For this reason, we recommend using a Linux platform.


  1. Create new directory.
  2. Execute rhc setup --server=[url] (make sure to use https if applicable). Enter your username/password if prompted.
  3. Execute rhc app create [appname] ruby-1.9 postgresql-9.2. Behind the scenes it will uses the REST API to communicate with OSE to create the application then perform a git clone to clone the directory locally. Once this is complete, a list of the application information will be shown. To show this information again, use rhc show-app [appname].
  4. From the parent directory (one directory up from your application directory), execute rails new [appname]. Say yes to any files it wants to overwrite.
  5. Adjust your Gemfile so the source uses http not https, i.e. source http://rubygems.org. Uncomment gem 'therubyracer'. Execute bundle install to verify this worked.
  6. At this point, you can push to OSE and verify things worked. Do the following: 
    git add .
    git commit -am "[message]"
    git push
    Verify this worked properly by navigating to your application landing page ([appname]-[namespace].[host] or you can follow the link from the web console).
  7. Scaffold your first entity. Execute rails generate scaffold [entityname] [fieldname:type]. We will use SQLite for now.
  8. Next we will add an action hook to run rake db:migrate as part of the deploy step. In the .openshift/action_hooks,create a file called deploy and add the following lines: 
    pushd ${OPENSHIFT_REPO_DIR} > /dev/null
    echo "Executing rake db:migrate for RAILS_ENV=${RAILS_ENV:-production} and RAILS_DB=${RAILS_DB}..."
    bundle exec rake db:migrate RAILS_ENV=${RAILS_ENV:-production}
    popd > /dev/null
  9. Make sure that git captures the file permissions with the following command: git update-index --chmod=+x [path/to/file]
  10. Modify your routes to add: root :to => "[entities]#index" where entities is the plural of your entityname. Delete public/index.html. This will set the default route to be the entity page.
  11. Push using git. You should see rake db:migrate execute as part of your workflow. Verify this worked properly by navigating to your application page. Test CRUD for your first entity using SQLite!

    12. To facilitate debugging, you can set the environment to be in development mode using: rhc env set RAILS_ENV=development. This will give greater debugging information when you do rhc tail.


  12. Now we will convert this to use PostreSQL. Start by adding gem 'pg' to your Gemfile and running bundle install.
  13. Change your config/database.yml file to the following:

    If you copy and paste the below configuration make sure the indenting is correct as per yml standards.


    sqlite: &sqlite
  adapter: sqlite3
  database: db/<%= Rails.env %>.sqlite3

mysql: &mysql
  adapter: mysql2
  database: "<%=ENV['OPENSHIFT_APP_NAME']%>"
  username: "<%=ENV['OPENSHIFT_MYSQL_DB_USERNAME']%>"
  password: "<%=ENV['OPENSHIFT_MYSQL_DB_PASSWORD']%>"
  host: <%=ENV['OPENSHIFT_MYSQL_DB_HOST']%>
  port: <%=ENV['OPENSHIFT_MYSQL_DB_PORT']%>

postgresql: &postgresql
  adapter: postgresql
  database: <%=ENV['OPENSHIFT_APP_NAME']%>
  username: <%=ENV['OPENSHIFT_POSTGRESQL_DB_USERNAME']%>
  password: <%=ENV['OPENSHIFT_POSTGRESQL_DB_PASSWORD']%>
  host: <%=ENV['OPENSHIFT_POSTGRESQL_DB_HOST']%>
  port: <%=ENV['OPENSHIFT_POSTGRESQL_DB_PORT']%>
  min_messages: ERROR
  reconnect: false

defaults: &defaults
  pool: 5
  timeout: 5000
  encoding: utf8
  <<: *<%= ENV['RAILS_DB'] || "sqlite" %>

development:
  <<: *defaults

test: &test
  <<: *defaults

production:
  <<: *defaults
    This also requires an environment variable to be set: rhc env set RAILS_DB=postgresql
  14. Push using git and verify this worked properly by navigating to your application landing page.
  15. Next, we will add a better look and feel using Bootstrap. For the purposes of this demo, just link the CDN maintained css and js files by adding the following to application.html.erb: 
    <!-- Latest compiled and minified CSS -->
<link rel="stylesheet" href="//netdna.bootstrapcdn.com/bootstrap/3.1.1/css/bootstrap.min.css">

<!-- Optional theme -->
<link rel="stylesheet" href="//netdna.bootstrapcdn.com/bootstrap/3.1.1/css/bootstrap-theme.min.css">

<!-- Latest compiled and minified JavaScript -->
<script src="//netdna.bootstrapcdn.com/bootstrap/3.1.1/js/bootstrap.min.js"></script>
  16. Modify your entity views to use the bootstrap styles (in app/views/[entity] directories). Some examples are available here: http://getbootstrap.com/css/
  17. For this demo, I created a nested object in Rails. This is fairly complex. This guide summarizes it but some additional changes will probably have to be made to make it work: https://gist.github.com/jhjguxin/3074080. To add a child entity, you can use the following scaffolding syntax: rails g scaffold [childentity] [fields:types] [parent:references]
  18. To add login, follow the steps in this tutorial: http://guides.railsgirls.com/devise/. One correction: in step 6, I had to use before_action not before_filter.
  19. To add Jenkins, I used the UI. First, I added a new Jenkins server. Then, I went to my existing application and clicked Enable Jenkins. All done!
